## Directory Purposes

**Project Root:**
- Purpose: Portfolio website entry point and SEO configuration
- Contains: Main HTML entry point, domain configuration, metadata files
- Key files: `index.html`, `CNAME`, `robots.txt`, `sitemap.xml`

**assets/:**
- Purpose: Central repository for all static resources
- Contains: Stylesheets, JavaScript, images, PDFs
- Served as-is by web server (no processing)

**assets/css/:**
- Purpose: Stylesheet storage
- Contains: Main styles and third-party carousel library
- Key files: `styles.css` (custom styles), `swiper-bundle.min.css` (library)

**assets/js/:**
- Purpose: JavaScript behavior code
- Contains: Custom interactivity and third-party libraries
- Key files: `main.js` (custom), `swiper-bundle.min.js` (library)

**assets/img/:**
- Purpose: Image asset storage
- Contains: Profile photos, project screenshots, certificate images
- Organized in subdirectories by project type (frontend_project, fullstack_project)

**assets/pdf/:**
- Purpose: Resume and portfolio document downloads
- Contains: Multiple versions of resume/portfolio PDFs
- Accessible via download links in HTML

**qr-profile/:**
- Purpose: Standalone digital business card application
- Contains: Self-contained HTML with embedded styles and JavaScript
- Use case: QR code scanning, mobile contact sharing

**.planning/codebase/:**
- Purpose: GSD codebase analysis and planning documents
- Contains: Architecture and structure documentation
- Auto-generated by development tools

## Naming Conventions

**Files:**
- HTML: lowercase, dashes for spacing (`index.html`)
- CSS: lowercase, dashes for spacing (`styles.css`, `swiper-bundle.min.css`)
- JavaScript: lowercase, dashes for spacing (`main.js`, `swiper-bundle.min.js`)
- Images: snake_case or descriptive names (`prefil_new.png`, `todoWebApp.png`)
- PDFs: PascalCase with version numbers (`NaveenBishtResume5.0.pdf`)

**Directories:**
- lowercase with hyphens for multi-word names (`qr-profile`, `frontend_project`)
- semantic naming indicating content type

**HTML Classes and IDs:**
- BEM-like naming: `block__element--modifier`
- Examples: `nav__logo`, `nav__menu`, `nav__link`, `skills__content`, `skills__close`, `contact__form`
- State classes: `active-link`, `show-menu`, `show-scroll`, `dark-theme`, `active-modal`
- Prefixes indicate component: `nav__*` for navigation, `skills__*` for skill sections

**CSS Variables:**
- Prefixed with `--`: `--header-height`, `--hue-color`, `--first-color`, `--text-color`
- Grouped by purpose: colors, fonts, sizing, spacing, z-index
- Dark theme variants use same variable names with body.dark-theme selector

## Where to Add New Code

**New Feature:**
- Primary code: Add `<section>` to `index.html` with unique id
- JavaScript handlers: Add event listeners to `/assets/js/main.js`
- Styling: Add styles to `/assets/css/styles.css`
- Images: Place in `/assets/img/` (create subdirectory if project-specific)

**New Component/Module:**
- Implementation: Define as HTML `<section>` or `<div>` with BEM classes in `index.html`
- Interactivity: Add JavaScript event listeners in `/assets/js/main.js`
- Styling: Create `.component__element` class in `/assets/css/styles.css`
- Follow existing pattern: Look at `skills` or `qualification` sections for patterns

**Utilities/Reusable Code:**
- Shared helpers: Extract to separate function in `/assets/js/main.js`
- No module system (direct DOM manipulation)
- Keep functions focused and named descriptively

**New Project Showcase:**
- Create screenshot image in `/assets/img/[project_type]/`
- Add `<div>` with class `grid-item` to `.project__container` in `index.html`
- Link to project and add image with alt text
- Update project showcase title/subtitle if needed

**Business Card Updates:**
- Edit `/qr-profile/index.html` directly (self-contained)
- Update contact information in contact-row elements
- Update vCard data in `downloadVCard()` function
- Update profile image path if new photo added

## Future Organization Considerations

**If adding more features:**
1. Consider splitting `assets/js/main.js` into modules (e.g., `navigation.js`, `theme.js`, `forms.js`)
2. Consider separating component styles in CSS (e.g., `components.css`, `utilities.css`)
3. Consider build process (Webpack, Vite) if complexity increases
4. Consider asset optimization (image compression, CSS/JS minification)

**For easier maintenance:**
- Keep one section per major feature (skills, projects, qualification)
- Use consistent class naming (BEM) throughout
- Document complex JavaScript behaviors
- Keep responsive breakpoints consistent with CSS variables
